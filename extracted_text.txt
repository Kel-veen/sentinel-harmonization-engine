3.1 INTRODUCTION
This chapter presents the methodological approach adopted in the design and development of the proposed system for harmonizing user privileges across multi-cloud platforms. The chapter provides a detailed explanation of the research design, system architecture, data collection procedures, data modeling techniques, algorithm design, system workflow, and evaluation methods.
The primary objective of this study is to develop a Python-based system capable of analyzing Identity and Access Management (IAM) configurations from AWS and Azure environments, identifying inconsistencies in user privileges, and generating harmonized recommendations that improve security and governance.
Unlike purely theoretical studies, this research emphasizes practical system development, ensuring that the proposed solution can be implemented and evaluated in a controlled environment.

It addresses a real-world cybersecurity problem
It involves design, development, and validation of a system

3.4 PROPOSED SYSTEM ARICHITECTURE
The system adopts a three-layer architecture, designed to ensure modularity, scalability, and clarity of operations.
Figure 10: System architecture (Created using ChatGPT)
3.4.2 EXPLANATION OF ARCHITECTURE
1. Data Ingestion Layer
This layer is responsible for collecting IAM data from both AWS and Azure environments. It processes JSON files and extracts relevant entities such as users, roles, permissions, and policies.
Privilege Analysis Layer
This is the core analytical component of the system. It converts raw data into a unified format and performs comparisons to detect inconsistencies, redundant permissions, and potential risks.
Harmonization Layer
This layer ensures consistency across platforms by mapping equivalent roles and generating actionable recommendations. It also produces reports that can be used for decision-making.

3.6 DATA MODELING AND NORMALIZATION
3.6.1 PROBLEM CONTEXT
AWS and Azure implement IAM using fundamentally different models:
AWS → Policy-based
Azure → Role-based
This creates incompatibility when attempting cross-platform analysis.

3.7 SYSTEM ALGORITHM
3.7.1 ALGORITHM STEPS
Load datasets
Normalize IAM data
Compare privileges
Detect inconsistencies
Identify excessive permissions
Map equivalent roles
Generate recommendations
3.7.2 LOGIC EXPLANATION
If a user has different permissions across platforms → Inconsistency detected
If a user has more permissions than required → Excess privilege risk
If roles differ but represent similar access → Mapping required
3.7.3 PSEUDOCODE
BEGIN
LOAD AWS_data
LOAD Azure_data
NORMALIZE datasets
FOR each user IN dataset:
    COMPARE permissions
    
    IF permissions mismatch:
        FLAG inconsistency
        
    IF permissions exceed requirement:
        FLAG excess privilege
        
    MAP equivalent roles
GENERATE recommendations
OUTPUT report
END
